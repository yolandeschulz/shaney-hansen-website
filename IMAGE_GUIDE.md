# Image Guide

The photography is the hero of this website — but only if the site stays fast. Follow these rules for every image.

## The one rule that matters most

**Never upload raw, full-resolution shoot files.** Always export a web version first.

## Export settings

| Use | Width (longest side) | Format | Target file size |
|---|---|---|---|
| Hero images (page tops) | 2000–2400 px | JPG (or WebP) | 400–800 KB |
| Gallery images | 1600 px | JPG (or WebP) | 200–500 KB |
| Cover images (category cards) | 1200 px | JPG (or WebP) | 150–400 KB |
| Small/inline images | 800–1200 px | JPG (or WebP) | under 200 KB |

In Lightroom: Export → JPEG → Quality 75–80 → Resize to fit long edge → sharpen for screen.

## File naming

Use descriptive, lowercase, hyphenated names — no spaces, no client surnames:

- Good: `wedding-first-dance-golden-light.jpg`, `newborn-sleeping-neutral-wrap.jpg`
- Avoid: `IMG_4302.JPG`, `Smith Family Final 12.jpg`

## Where images go

Each category has its own folder under `public/images/`:

`front-page`, `weddings`, `couples`, `family`, `maternity`, `newborn`, `cake-smash`, `school`, `pre-school`, `sport`, `corporate`, `studio`, `styled`, `social-media`, `team-up-athletics`, `creative-direction`, `general`

Upload through Pages CMS (it saves to the right folder) or place files directly in the folder on GitHub.

## Alt text (required for every image)

Alt text is a short natural sentence describing the photo. It helps Google and visitors using screen readers.

- "Family photography session in soft natural light"
- "Bride and groom wedding portrait by Shaney Hansen Photography"
- "Cake smash birthday photography session with styled set"
- "Corporate headshot photography by Shaney Hansen Photography"

Don't keyword-stuff, don't include client names or private details.

## Choosing cover images

Pick images with: strong emotion, good lighting, a clear subject, timeless feel and warmth. Avoid busy backgrounds, near-duplicates and anything that weakens the premium feel.

## How many images per gallery

12–24 strong images per gallery is ideal. Curate ruthlessly — a tight gallery of excellent work reads as more premium than a long gallery with filler.

## Keeping the site fast

- Stick to the file-size targets above
- Prefer WebP where easy (smaller than JPG at the same quality)
- Replace the placeholder `.svg` files as real images arrive, and update the image references in the gallery entries
- If a page ever feels slow, the first suspect is an oversized image
